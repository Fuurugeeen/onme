1. プロダクト概要

本プロダクトは、SNSやショート動画に時間を奪われがちな学生を対象にした、自己理解のためのAIコーチングサービスである。

現代の学生は常に外部からの刺激（通知、おすすめ動画、他者の投稿）にさらされ、「自分に目を向ける時間」を持てないでいる。本プロダクトは、比較のない安全な場所で、AIエージェントと対話しながら自分の強み・弱み・特性を発見していくことを目的とする。

特徴：
	•	AIエージェントがユーザーの「考え方の型」を学習し、パーソナライズされたコーチングを提供
	•	AIエージェント同士がユーザーの代わりに交流し、匿名で知見を収集・共有

⸻

2. 想定ユーザー

メインターゲット
	•	SNS・ショート動画で気づけば時間を溶かしてしまう学生
	•	他人と比較して落ち込みやすい学生
	•	自分の強みや特性が分からず、将来に漠然とした不安がある学生
	•	自己分析ツールやキャリア教材を途中でやめがちな学生

非ターゲット
	•	明確な目標を持ち、自走できている学生
	•	SNS上での競争・承認欲求を楽しんでいる層

⸻

3. 解決したい課題

根本課題：外部刺激に時間を奪われ、自分に目を向ける余裕がない

	1.	自分の価値観・強み・弱みが言語化できない
	2.	SNSの比較文化で自己肯定感が下がる
	3.	「自分と向き合う」時間・場所がない
	4.	大人（先生・親）との面談は心理的に重い
	5.	一人で内省しても続かない・深まらない

⸻

4. プロダクトコンセプト

キーワード
	•	比較しない場所
	•	自分を知る
	•	AIが寄り添う

基本思想
	•	他人との比較・偏差値・いいね数は一切使わない
	•	「昨日の自分」との比較のみを評価軸にする
	•	SNSの代替ではなく、SNSから離れる「避難所」を提供
	•	AIが自分を映す鏡となり、自己理解を促進する

プロダクトの核心的価値
	•	AIエージェントが「あなたの考え方の型」を把握し、コーチングに活用
	•	AIエージェント同士が交流し、似た悩みを持つ人の知見を匿名で届ける
	•	結果として、孤独ではなく「見えない仲間」がいる安心感を提供

5. 機能設計

5.1 初期オンボーディング
	•	所要時間：10〜15分
	•	内容：
	•	簡単な質問（選択式＋短文）
	•	価値観・興味・ストレス耐性の推定
	•	診断結果はラベル化せず、内部パラメータとして保持
	•	質問設計：AIが最適な質問を動的に生成

5.2 デイリーAIコーチ
	•	1日1回、5分以内で完結
	•	AIのトーン：フラットな友達風（対等でカジュアルな会話）
	•	フロー：
	1.	今日の状態確認（気分・余裕度）
	2.	昨日の行動の簡単な振り返り
	3.	今日やる「1タスク」の提示

例：
	•	「英語を30分」ではなく「英単語10個を見る」
	•	必ず 失敗しにくい粒度 に分解

タスク生成ロジック：
	•	事前テンプレート＋AI動的生成のハイブリッド方式
	•	テンプレートで品質を担保しつつ、AIがユーザーに合わせてカスタマイズ

対象タスクカテゴリ：
	•	勉強・学習（英語、読書、資格勉強など）
	•	生活習慣（早起き、睡眠、整理整頓など）
	•	運動・健康（ストレッチ、散歩など）
	•	自己探索・趣味（新しいことに挑戦など）

5.3 行動ログと可視化
	•	記録するのは以下のみ：
	•	実行した / しなかった
	•	主観的な負荷（軽い〜重い）
	•	グラフは
	•	継続日数
	•	行動実行率

5.4 フィードバック
	•	AIが以下を調整：
	•	タスクの難易度
	•	提示頻度
	•	声かけのトーン

5.5 ユーザー学習機能（パーソナライズAI）

目的：
	•	ユーザーの「考え方の型」を把握し、その人に最適なコーチングを提供
	•	使えば使うほど、自分を理解してくれるAIになる

■ 収集データ

| データ種別 | 収集タイミング | 例 |
|-----------|--------------|-----|
| 会話内容 | 毎回の対話 | 発言、回答、質問への反応 |
| 感情・状態 | デイリーチェックイン | 今日の気分（5段階）、余裕度 |
| 行動ログ | タスク完了時 | 実行/未実行、主観的負荷、時間帯 |
| 選択傾向 | オンボーディング・日常 | 選んだ選択肢、迷った時間 |
| 振り返り内容 | 週次振り返り | 良かったこと、難しかったこと |

■ 分析・抽出方法（Gemini API活用）

Step 1: 会話ごとの特徴抽出
```
【プロンプト例】
以下の会話から、ユーザーの特徴を抽出してJSON形式で返してください：
- 思考スタイル（論理的/感覚的）
- 意思決定パターン（即決型/熟考型）
- モチベーション源（達成/承認/成長/安定）
- ストレス反応（回避/対峙/相談）
- 表出された価値観キーワード

会話：{conversation_history}
```

Step 2: プロファイル更新（差分マージ）
```
【プロンプト例】
既存プロファイルと新しい観察結果を統合してください。
矛盾がある場合は、新しい情報を優先しつつ、
変化の傾向として記録してください。

既存プロファイル：{current_profile}
新しい観察：{new_observations}
```

■ 更新ロジック

| タイミング | 処理内容 |
|-----------|---------|
| 毎会話後 | 特徴抽出 → 一時バッファに保存 |
| 1日の終わり | バッファを集約 → プロファイル更新 |
| 週次 | 傾向分析 → 長期パターン更新 |
| 月次 | 成長・変化の検出 → インサイト生成 |

■ コーチングへの活用

| プロファイル項目 | 活用例 |
|-----------------|--------|
| thinking_style | 論理的な人→データで説明、直感的な人→イメージで伝える |
| motivation_drivers | 成長重視→「新しいスキルが身につく」と伝える |
| stress_response | 回避傾向→プレッシャーをかけない声かけ |
| best_time | 午後型→午後にタスク通知を送る |
| values | 「自由」重視→選択肢を多めに提示 |
| strengths_discovered | 定期的に強みをフィードバック |

■ 活用例

例1: 熟考型ユーザーへの対応
```
通常：「今日やるタスクを決めよう！」

　　　　明日の朝までに決めてくれればOKだよ」
```

例2: 承認欲求が低いユーザーへの対応
```
通常：「すごい！よく頑張ったね！」
承認低め：「予定通り完了したね。次は何をやろうか？」
```

例3: 強みの発見フィードバック
```
「最近の会話を見てて思ったんだけど、
　君は人の話をじっくり聞いて考えるタイプだね。
　これって『傾聴力』っていう強みだと思うよ。」
```

5.6 AIエージェント交流（コーチング品質向上システム）

目的：
	•	ユーザーには見えない裏側で、AIエージェント同士が学習
	•	効果的だったコーチング手法を共有し、全体の品質を向上
	•	「比較しない」コンセプトを維持しつつ、AIの質を高める

■ ユーザー体験
	•	ユーザーには何も見えない
	•	ただ、使い続けると「AIの質が上がっている」と感じる
	•	「このAI、私のこと分かってきたな」という実感

■ 共有される知見（匿名化・抽象化済み）

| 知見タイプ | 内容 | 活用方法 |
|-----------|------|---------|
| 効果的な問いかけ | 「この問いで自己理解が深まった」 | 類似プロファイルに適用 |
| 成功アプローチ | 「熟考型には選択肢を絞ると効果的」 | パターンとして学習 |
| 失敗パターン | 「この声かけは離脱につながった」 | 回避ルールとして学習 |
| 効果的な言い換え | 「弱み→強みへのリフレーミング例」 | 表現の改善に活用 |

■ 技術的な実装フロー

```
[ユーザーAとの会話]
        ↓
[効果測定：7日継続率を計測]
        ↓
[効果的だったアプローチを抽出]
  - どの問いかけの後に継続率が上がったか
  - どのタイミングの介入が効果的だったか
        ↓
[匿名化・抽象化]
  - ユーザー固有情報を削除
  - プロファイルタイプのみ残す
  - コーチング手法をパターン化
        ↓
[コーチング知見DB]
  - 効果的な問いかけ集
  - 失敗パターン集
  - プロファイル別ベストプラクティス
        ↓
[ユーザーBのAIエージェント]
  - 類似プロファイルの知見を参照
  - コーチングに反映
```

■ 効果測定指標（複合スコア）

効果スコア = 継続率(40%) + 会話深度(40%) + 自己理解スコア(20%)

【1. 継続率（客観・自動計測）】
	•	測定：7日後もログインしているか
	•	計算：継続=1.0、離脱=0.0

【2. 会話深度（AI判定）】
	•	測定：会話終了後にGemini APIで判定
	•	判定基準（各0.0〜1.0）：

```
【プロンプト例】
以下の会話を分析し、3つの観点でスコアを返してください：

1. 自己開示度（0.0〜1.0）
   - 個人的な感情、経験、悩みをどれだけ話したか
   - 0.0: 表面的な返答のみ
   - 1.0: 深い感情や過去の経験を共有

2. 具体性（0.0〜1.0）
   - 抽象的な話から具体的なエピソードに深まったか
   - 0.0: 「なんとなく不安」レベル
   - 1.0: 「先週の〇〇の時に△△と感じた」レベル

3. 気づきの表明（0.0〜1.0）
   - 「分かった」「気づいた」「そうかも」などの発言
   - 0.0: 気づきの発言なし
   - 1.0: 明確な気づき・発見の表明

会話：{conversation}
```

	•	計算：(自己開示度 + 具体性 + 気づき表明) / 3

【3. 自己理解スコア（主観・ユーザー回答）】
	•	測定：週1回、ユーザーに質問
	•	質問例：「今週、自分のことを前より理解できた実感はありますか？」
	•	回答：1〜5段階 → 0.0〜1.0に正規化

【効果スコアの活用】
	•	スコア0.7以上 → 「効果あり」と判定
	•	特定アプローチ適用後のスコア変化を計測
	•	有意に効果があったアプローチを知見DBに追加

【効果判定の例】
```json
{
  "approach": "動物に例える問いかけ",
  "before_avg_score": 0.52,
  "after_avg_score": 0.71,
  "lift": 0.19,
  "sample_size": 45,
  "significant": true
}
```

■ 知見DB実装方針
	•	MVPから自動構築を実装
	•	初期データ：コーチング理論に基づくシードデータを投入
	•	自動学習：ユーザー1人目から知見を蓄積開始
	•	適用条件：サンプル数が少ない段階はシードデータを優先

■ 知見DB構造

```sql
-- コーチング知見テーブル
coaching_insights (
  id: UUID,
  insight_type: ENUM('effective_question', 'success_approach',
                     'failure_pattern', 'reframe_example'),
  content: JSONB,           -- 問いかけ内容、アプローチ詳細など
  target_profile: JSONB,    -- 適用対象プロファイルタイプ
  context: VARCHAR,         -- 適用コンテキスト（オンボーディング等）
  effect_metric: JSONB,     -- 効果測定結果
  sample_size: INT,         -- サンプル数
  source: ENUM('seed', 'learned'),  -- シード or 学習済み
  created_at: TIMESTAMP,
  updated_at: TIMESTAMP
)

-- 知見適用ログ（効果測定用）
insight_applications (
  id: UUID,
  user_id: UUID,
  insight_id: UUID,
  applied_at: TIMESTAMP,
  retention_day7: BOOLEAN,  -- 7日後も継続しているか
  measured_at: TIMESTAMP
)
```

■ 自動学習フロー

```
[毎会話後]
  → 会話内容をGemini APIで分析
  → 使用したアプローチを記録

[7日後（バッチ処理）]
  → 継続しているか判定
  → アプローチごとの継続率を集計
  → 統計的に有意なパターンを知見DBに追加/更新

[知見適用時]
  → ユーザープロファイルに合致する知見を検索
  → sample_size が大きい知見を優先
  → コーチングに反映
```

■ プライバシー保護
	•	個人を特定できる情報は一切共有しない
	•	プロファイルタイプ（カテゴリ）のみを使用
	•	統計的に有意なサンプル数が集まってから知見化

12. 本プロダクトの本質

本プロダクトは「正解を与えるAI」ではない。
SNSの比較文化から離れ、自分と静かに向き合う「避難所」である。

AIエージェントは：
	•	あなたの考え方の型を学び、鏡のように自分を映し出す存在
	•	他のユーザーのAIと交流し、「見えない仲間」の知見を届ける存在

最終ゴールは、自分の強み・弱み・特性を理解し、
AIがなくても「自分はこういう人間だ」と言えるようになること。
